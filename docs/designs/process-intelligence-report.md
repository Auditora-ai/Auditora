## Vision

### 10x Check
A client finishes a 1-hour session. 90 seconds later, their consultant shares a link. The client opens it on their phone. They see a Process Maturity Score of 67/100. Below it, a SIPOC radar chart. A visual process map with 14 activities across 3 roles. Tap an activity — full SOP expands. Risk heat map. Automation opportunities. "Recommended Next Session: Deep Dive into Supplier Integration." The VP forwards the link to their boss. They think: "This came from ONE hour? We spent 6 months with Deloitte and got less."

### Platonic Ideal
The perfect process intelligence report feels like having a senior BPM consultant walk you through everything they observed. Not a list of findings. A narrative with visual hierarchy that guides the eye — the most important insight is impossible to miss. Equally powerful on laptop or in a board meeting PDF.

## Reviewer Concerns (from spec review)
1. **SIPOC data source:** Resolved — derive from knowledge snapshot confidence scores, not a new AI pipeline. Zero latency, no new tokens.
2. **Performance budget:** New pipeline calls are only complexity_score (~500 token output). SIPOC is deterministic compute. 90-second target is achievable since 4 existing pipelines already run in parallel and complexity is lightweight.
3. **Share link security:** Token-based (existing pattern). No expiry in Phase 1. Revocation by regenerating token.
4. **Phasing:** All 7 features ship together — they're mostly presentation changes over existing data. The only new pipeline call is complexity_score (already exists). Effort is L not XL.
5. **Full technical design:** See implementation plan in Claude Code plan file.
