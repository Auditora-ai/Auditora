## Vision

### 10x Check
Transform the process detail page from a form-based lifecycle tracker into a **Process Intelligence Workspace**. The BPMN diagram becomes the navigation layer — clicking elements reveals their context (RACI, risks, SOP, inputs/outputs). Health badges on nodes show completeness at a glance. One-click client-ready exports. Version diff overlays directly on the diagram. The feeling: a precision instrument for BPM consultants.

### Platonic Ideal
Open a process and instantly see the whole picture. The diagram fills the screen. A subtle health bar shows "72% documented — RACI missing for 3 activities." Click a task node, sidebar shows its RACI, risks, and SOP. Fill in the missing RACI. Health bar updates. Click Export in the header — a professional HTML report generates. The tool makes the consultant look like the best in the room.

## Accepted Scope (added to this plan)

### Core Redesign
- Diagram-centric layout: BPMN takes 65% of viewport, sidebar 35% (sidebar collapsible)
- Rewrite ProcessDetailView (~1600 lines) into modular component tree
- Remove phase-based dashboard (Contexto/Captura/Modelo/Analisis cards)
- Status bar header with process name, health ring, export, edit actions

### Expansions
- Contextual sidebar that reacts to diagram element selection (via bpmn-js selection events)
- Green/yellow/red health badges on BPMN nodes using bpmn-js Overlays API (viewport-only rendering for performance)
- Prominent export button in header (reuse existing export-book, generates HTML report)
- Version diff toggle overlay: compare current BPMN XML vs selected previous version, highlight added/removed/modified elements via bpmn-js Overlays with colored borders
- Mini-map via bpmn-js minimap module (already available) + keyboard shortcuts (Cmd/Ctrl+E export, Cmd/Ctrl+S save, Escape deselect, +/- zoom) with preventDefault for browser conflicts
- Clickable maturity flow in header: Draft -> Mapped -> Validated -> Approved (processStatus field, free-form String but validated in UI)

## Implementation Notes

### Data Model
- **Per-node health** is computed client-side from already-loaded data: process includes RACI entries (by activityName), risks (by nodeId), and sessions with diagramNodes. No new DB queries needed. Health formula: green = has RACI + has risk assessment, yellow = partial, red = missing both.
- **Sidebar per-click data** comes from the process's existing include tree (RACI entries, risks, diagram nodes). No new API endpoint needed — all data loaded at page level in the server component.
- **Maturity transitions** validated in UI: Draft -> Mapped (requires bpmnXml), Mapped -> Validated (requires RACI), Validated -> Approved (manual). PATCH to existing `/api/processes/[processId]`.

### Rendering Approach
- **Health badges:** bpmn-js Overlays API (`overlays.add(elementId, { html: badgeHtml, position: { top: -10, right: -10 } })`). Updated on data change, removed/re-added on viewport change for large diagrams.
- **Version diff:** Parse both BPMN XMLs, extract element IDs from each, compute set difference (added/removed/modified). Apply colored overlays via bpmn-js Overlays. Toggle from DiagramToolbar button.
- **Sidebar element types:** Tasks/SubProcesses show full context (RACI, risks, SOP). Gateways show decision logic. Events show triggers/outputs. Sequence flows show conditions. Lanes show participant info.

### Phased Rollout
- **Phase 1:** Component decomposition (split monolith, no new features, same visual output)
- **Phase 2:** Diagram-centric layout + sidebar tabs + header redesign
- **Phase 3:** Contextual sidebar + health badges + export button + maturity flow + keyboard shortcuts
- **Phase 4:** Version diff overlay + mini-map

### Mobile/Tablet
- Sidebar collapses to bottom sheet (draggable, like Google Maps mobile)
- Diagram takes full width above the sheet
- Export and actions accessible from header (always visible)
