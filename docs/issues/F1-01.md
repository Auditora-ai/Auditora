F1-01: Extraer diagramador BPMN como lib compartida (process-engine)

## Contexto
El diagramador BPMN vive dentro de `modules/meeting/` (hooks/useBpmnModeler.ts, lib/bpmn-builder.ts, lib/bpmn-validator.ts, lib/bpmn-colors.ts, lib/patterns.ts, context/LiveSessionContext.tsx). También se usa parcialmente en `modules/radiografia/` y `modules/process-library/`. Esto crea duplicación y acoplamiento — el scan no debería depender del módulo de videollamada.

## Objetivo
Crear `packages/process-engine/` como librería compartida con todo lo relacionado a BPMN: modeler, builder, validator, colors, patterns. Los módulos `meeting/`, `radiografia/` y `process-library/` la consumen como dependencia.

## TDD — Tests primero

### Tests a escribir ANTES de refactorizar:
1. **`bpmn-builder.test.ts`** — Dado un array de DiagramNodes, genera XML BPMN válido con ELK layout. Casos: start→task→end, gateway con 2 salidas, proceso con lanes.
2. **`bpmn-validator.test.ts`** — Dado XML BPMN, detecta violaciones: start sin end, gateway sin 2+ salidas, nodos huérfanos, ciclos. Retorna array de violations.
3. **`bpmn-colors.test.ts`** — Dado tipo de nodo (TASK, START_EVENT, etc.), retorna colores correctos según DESIGN.md (border + fill).
4. **`node-type-mapping.test.ts`** — Mapeo AI output (camelCase: `startEvent`, `userTask`) → Prisma enum (`START_EVENT`, `USER_TASK`) vía `toNodeType()`. Casos edge: null, undefined, tipo desconocido.
