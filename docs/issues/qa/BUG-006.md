## BUG-006: SessionProvider loaded never true for unauthenticated users
**Severity:** High
**Component:** apps/saas/modules/auth/components/SessionProvider.tsx
**Found by:** QA Agent #05
**Date:** 2026-04-02

### Steps to Reproduce
1. Access the application without being authenticated
2. Observe any component that gates rendering on SessionProvider's loaded state
3. Components remain in loading/hung state indefinitely

### Expected Behavior
loaded becomes true once session status is resolved (even if session is null)

### Actual Behavior
loaded stays false when session is null, because the loaded flag is only set when session is truthy

### Code Reference
File: apps/saas/modules/auth/components/SessionProvider.tsx
Lines: 12-18

### Suggested Fix
Set loaded to true regardless of whether session is null or truthy — only keep it false during the initial fetch

### Regression Risk
High — affects all components that depend on SessionProvider's loaded state
