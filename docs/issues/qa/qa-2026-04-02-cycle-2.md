## 3. New Bugs Found (Cycle 2)

### 3.1 BUG-022 — TypeScript: `name: null` incompatible with Prisma Organization schema
**Severity:** 🟠 High  
**File:** `apps/saas/app/api/organization/profile/route.ts:66`

The fix for BUG-011 added `companyName: z.string().nullable().optional()` in the Zod schema, allowing `null` to pass through to `name: null` in Prisma's `organization.update()`. But `Organization.name` is `String` (non-nullable) in the Prisma schema.

**Fix:** Change Zod to `z.string().min(1).optional()` (no null allowed) — aligns with Prisma constraint.

---

### 3.2 BUG-023 — TypeScript: LoginForm `mode` default `"***"` not in Zod union
**Severity:** 🟠 High  
**File:** `apps/saas/modules/auth/components/LoginForm.tsx:77`

The ternary `authConfig.enablePasswordLogin ? "password" : "***"` uses `"***"` as the fallback when password login is disabled, but only `"magic-link"` | `"password"` are valid.

**Fix:** Change `"***"` to `"magic-link"`:
```typescript
mode: authConfig.enablePasswordLogin ? "password" : "magic-link",
```

No runtime impact (currently `enablePasswordLogin: true`), but TypeScript validation fails.

---

### 3.3 BUG-024 — Hardcoded Spanish `"Inicio"`/`"Fin"` in `sipoc-to-nodes.ts`
**Severity:** 🟡 Medium  
**File:** `apps/saas/modules/radiografia/lib/sipoc-to-nodes.ts:27,82`

This file was missed in the BUG-012 fix sweep. When SIPOC has no `inputs[0]?.name`, the start event gets label `"Inicio"`. When no `outputs[0]?.name`, end event gets `"Fin"`. Affects English/French/German users.

**Fix:** Change fallbacks to `""` (empty string), consistent with `bpmn-builder.ts`.

---

### 3.4 BUG-025 — Hardcoded Spanish labels in BPMN node workspace
**Severity:** 🟢 Low  
**Files:** `meeting/lib/node-display-config.ts`, `process-library/workspace/NodeContextPanel.tsx`

`node-display-config.ts` has labels `"Inicio"`, `"Fin"`, `"Tarea"`, etc. as static strings.  
`NodeContextPanel.tsx` has inline JSX `"Este evento inicia el proceso."` / `"Este evento finaliza el proceso."`.

**Fix:** Move labels to translation keys or use `useTranslations()` hook.
