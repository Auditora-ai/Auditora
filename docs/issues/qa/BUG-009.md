## BUG-009: Missing .min(1) on name validation in onboarding
**Severity:** High
**Component:** apps/saas/modules/onboarding/components/OnboardingAccountStep.tsx
**Found by:** QA Agent #05
**Date:** 2026-04-02

### Steps to Reproduce
1. Start the onboarding flow at the account step
2. Submit the form with an empty name field
3. Observe validation passes when it should fail

### Expected Behavior
Validation fails with an error requiring a non-empty name

### Actual Behavior
z.string() allows an empty string, so the form submits successfully with a blank name

### Code Reference
File: apps/saas/modules/onboarding/components/OnboardingAccountStep.tsx
Lines: 23-25

### Suggested Fix
Change z.string() to z.string().min(1) for the name field validation

### Regression Risk
Low — tightening validation constraint
