## BUG-002: Organization profile API uses first org instead of active org
**Severity:** Critical
**Component:** apps/saas/app/api/organization/profile/route.ts
**Found by:** QA Agent #05
**Date:** 2026-04-02

### Steps to Reproduce
1. User belongs to multiple organizations
2. Set active org to Org B
3. PUT to /api/organization/profile with Org B data
4. Data updates Org A instead

### Expected Behavior
Updates the currently active organization

### Actual Behavior
Updates orgs[0] regardless of which is active

### Code Reference
File: apps/saas/app/api/organization/profile/route.ts
Lines: 13-16 (getOrgId uses orgs?.[0]?.id)

### Suggested Fix
Use session.activeOrganizationId or accept organizationId as parameter

### Regression Risk
High — multi-org users will have data corruption
