## BUG-007: No input validation on organization profile PUT
**Severity:** High
**Component:** apps/saas/app/api/organization/profile/route.ts
**Found by:** QA Agent #05
**Date:** 2026-04-02

### Steps to Reproduce
1. PUT to /api/organization/profile with arbitrary unexpected fields in the request body
2. Include fields like "role": "admin" or other schema-external properties
3. Observe the request succeeds and arbitrary data passes to db.organization.update

### Expected Behavior
Request body is validated with Zod schema — only expected fields are accepted

### Actual Behavior
No Zod validation exists; arbitrary fields pass directly to db.organization.update

### Code Reference
File: apps/saas/app/api/organization/profile/route.ts
Lines: 37-63

### Suggested Fix
Add a Zod schema to validate the request body before passing to db.organization.update, whitelisting only allowed fields

### Regression Risk
Medium — affects organization profile update endpoint
