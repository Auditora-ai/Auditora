## BUG-005: Unsafe JSON.parse on riskResults
**Severity:** High
**Component:** apps/saas/app/api/scan/save/route.ts
**Found by:** QA Agent #05
**Date:** 2026-04-02

### Steps to Reproduce
1. Perform a scan where riskResults is stored as an object (not a string)
2. Attempt to save scan results via POST /api/scan/save
3. Server throws uncaught error on JSON.parse

### Expected Behavior
Scan results are saved regardless of whether riskResults is a string or object

### Actual Behavior
JSON.parse(scanSession.riskResults as string) throws when riskResults is already an object, causing a 500 error

### Code Reference
File: apps/saas/app/api/scan/save/route.ts
Lines: 52-53

### Suggested Fix
Add a type check before parsing: if typeof riskResults is "string" then JSON.parse, otherwise use the value directly

### Regression Risk
Medium — affects scan save endpoint
