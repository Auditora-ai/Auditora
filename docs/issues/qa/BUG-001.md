## BUG-001: Auth bypass in verifyOrganizationMembership — callers don't check null
**Severity:** Critical
**Component:** packages/api/modules/organizations/lib/membership.ts
**Found by:** QA Agent #05
**Date:** 2026-04-02

### Steps to Reproduce
1. Call any oRPC procedure that uses verifyOrganizationMembership
2. Pass an orgId the user doesn't belong to
3. Function returns null but caller proceeds

### Expected Behavior
Unauthorized error thrown

### Actual Behavior
Org data returned to unauthorized user

### Code Reference
File: packages/api/modules/organizations/lib/membership.ts
Lines: 3-17

### Suggested Fix
Throw an error inside verifyOrganizationMembership when membership is null, or have all callers check the return

### Regression Risk
High — affects all org-scoped oRPC procedures
