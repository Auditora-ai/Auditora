## BUG-023: TypeScript error — LoginForm `mode` default value `"***"` not in Zod union schema

**Severity:** High  
**Component:** `apps/saas/modules/auth/components/LoginForm.tsx` (line 77)  
**Found by:** QA Agent #05  
**Date:** 2026-04-02  
**Status:** 🔴 OPEN

---

### Steps to Reproduce

1. TypeScript compile: `pnpm --filter saas exec tsc --noEmit`
2. Observe error on line 74 of `LoginForm.tsx`

### Expected Behavior

No TypeScript errors. When `enablePasswordLogin` is `false`, the form mode should default to `"magic-link"`.

### Actual Behavior

```
modules/auth/components/LoginForm.tsx(74,3): error TS2322:
  Type '{ email: string; password: string; mode: "password" | "***"; rememberMe: true; }'
  is not assignable to ... 
  Type '"***"' is not assignable to type '"password" | undefined'.
```

### Root Cause

The form's `defaultValues` use a ternary:

```typescript
mode: authConfig.enablePasswordLogin ? "password" : "***",
```

The value `"***"` is used as a fallback when password login is disabled, but it is NOT in the Zod schema which only accepts `"magic-link"` | `"password"`. TypeScript correctly rejects this.

The intent was clearly to use `"magic-link"` as the fallback, but `"***"` was written by mistake.

### Code Reference

File: `apps/saas/modules/auth/components/LoginForm.tsx`  
Line: 77  

```typescript
// Zod schema (lines 46–58):
const formSchema = z.union([
  z.object({ mode: z.literal("magic-link"), ... }),
  z.object({ mode: z.literal("password"), ... }),
]);

// Default values (line 77):
mode: authConfig.enablePasswordLogin ? "password" : "***",
//                                                    ↑ WRONG: should be "magic-link"
```

### Suggested Fix

```typescript
mode: authConfig.enablePasswordLogin ? "password" : "magic-link",
```

### Production Impact

Currently `authConfig.enablePasswordLogin = true` in production, so the broken branch is never hit at runtime. However:
1. TypeScript compilation has an error
2. If password login is ever disabled in config, the form initializes with an invalid mode `"***"`, breaking the login form

### Regression Risk

Low at runtime (config currently has `enablePasswordLogin: true`), but High for type safety standards and potential future config changes.
