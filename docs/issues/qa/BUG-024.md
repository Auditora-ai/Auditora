## BUG-024: Hardcoded Spanish fallback labels `"Inicio"`/`"Fin"` in `sipoc-to-nodes.ts`

**Severity:** Medium  
**Component:** `apps/saas/modules/radiografia/lib/sipoc-to-nodes.ts` (lines 27, 82)  
**Found by:** QA Agent #05  
**Date:** 2026-04-02  
**Status:** 🔴 OPEN

---

### Steps to Reproduce

1. Run a scan that returns a SIPOC with no inputs defined
2. Observe the generated BPMN diagram start event has "Inicio" label
3. Repeat for empty outputs — end event will have "Fin"

### Expected Behavior

BPMN start/end event labels should either be:
- Empty string (no label) — consistent with the fix in `buildBpmnXml` (BUG-012 fix)
- Or locale-aware translation strings

### Actual Behavior

When SIPOC `inputs[0]?.name` is undefined, the start event label falls back to the hardcoded Spanish string `"Inicio"`. Similarly, when `outputs[0]?.name` is undefined, the end event gets `"Fin"`.

This produces Spanish-only BPMN output for non-Spanish-speaking users (English, French, German).

### Code Reference

File: `apps/saas/modules/radiografia/lib/sipoc-to-nodes.ts`  
Lines: 26–27 and 81–82  

```typescript
// Line 26-27:
const startLabel =
    sipoc.inputs[0]?.name || "Inicio";   // ← hardcoded Spanish

// Line 81-82:
const endLabel =
    sipoc.outputs[0]?.name || "Fin";     // ← hardcoded Spanish
```

### Context

BUG-012 already fixed the `bpmn-builder.ts` to not hardcode "Inicio"/"Fin". This file was missed in that fix sweep. It's in the scan/radiografia pipeline which generates BPMN from SIPOC data.

### Suggested Fix

Change fallbacks to empty string (matching the approach taken in `bpmn-builder.ts`):

```typescript
const startLabel = sipoc.inputs[0]?.name || "";
const endLabel = sipoc.outputs[0]?.name || "";
```

This is consistent with the updated `buildBpmnXml` behavior (empty = no label shown in diagram).

### Regression Risk

Low — only affects fallback labels when SIPOC has no inputs/outputs defined. The BPMN still generates correctly, only the label text is wrong. But this affects all non-Spanish users who run a scan with incomplete SIPOC data.
